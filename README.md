# CRMproekt — CRM для управления проектами, клиентами и расчётами

Веб-приложение: публичный сайт (услуги + форма заявки) и защищённая админ-панель
(заявки, клиенты, проекты, расчёт, справочник типов).

**Стек:** Python + Flask, база SQLite, фронтенд Bootstrap 5 + чистый JS (REST API).

Вход в админку: логин `admin`, пароль `admin`.

---

## 1. Запуск на своём компьютере (локально)

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Запустить
python app.py
```

Открыть в браузере: <http://localhost:5000>
База `database.db` создаётся автоматически при первом запуске.

## 3. Деплой на бесплатный сервер

Проект уже готов к деплою — есть `Procfile`, `wsgi.py`, `requirements.txt`, `runtime.txt`.
Запускающая команда (production-сервер gunicorn):

```
gunicorn wsgi:application --bind 0.0.0.0:$PORT
```

### Вариант А — Render.com (проще всего, рекомендую)
1. Зайти на <https://render.com>, войти через GitHub.
2. **New → Web Service** → выбрать ваш репозиторий.
3. Настройки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn wsgi:application --bind 0.0.0.0:$PORT`
   - Instance Type: **Free**.
4. Готово — получите адрес вида `https://crm-proekt.onrender.com`.

### Вариант Б — PythonAnywhere (лучший для SQLite — диск НЕ стирается)
1. <https://www.pythonanywhere.com> → бесплатный аккаунт.
2. **Web → Add a new web app → Flask → Python 3.x**.
3. Загрузить файлы (или `git clone` в консоли PythonAnywhere).
4. В **WSGI configuration file** указать импорт `from app import app as application`.
5. Reload — сайт по адресу `ВАШ_ЛОГИН.pythonanywhere.com`.

### Вариант В — Railway / Koyeb / Fly.io
Аналогично Render: подключаете GitHub-репозиторий, платформа сама видит `Procfile`.

> ⚠️ **Важно про базу на бесплатных хостингах.** На Render/Railway диск «эфемерный»:
> при каждом передеплое `database.db` обнуляется. Для учебной демонстрации это нормально.
> Если нужна **постоянная** база — берите **PythonAnywhere** (постоянный диск)
> или подключите внешнюю БД (PostgreSQL).

> Чтобы привязать свой домен — в настройках хостинга есть раздел **Custom Domain**,
> там укажете CNAME/A-запись, которую пропишете у регистратора домена.

## Структура проекта

```
app.py            — бэкенд Flask + REST API + создание БД
wsgi.py           — точка входа для production-сервера (gunicorn)
index.html        — весь фронтенд (публичный сайт + админка)
requirements.txt  — зависимости Python
Procfile          — команда запуска для хостинга
runtime.txt       — версия Python для хостинга
database.db       — база (создаётся автоматически, в git не хранится)
```

## Логика расчёта строки проекта

| Колонка | Обозн. | Формула |
|---|---|---|
| Количество | A1 | ручной ввод |
| Стоимость за ед. | A2 | ручной ввод |
| Сумма | A3 | A1 × A2 |
| Налог | A4 | A3 × ставка (5 / 7 / 10 / 22 %) |
| Себестоимость за ед. | A5 | ручной ввод |
| Сумма себестоимости | A6 | A1 × A5 + A4 |
| Прибыль | A7 | A3 − A6 |

Внизу таблицы расчёта — итоговая строка (сумма всех A7 по проекту).
Ставку налога можно менять прямо в ячейке «Налог».

## Разделы

- **Клиенты** — контактная информация, поиск (имя/телефон/заказы), карточка клиента
  с его проектами/заказами и отзывами.
- **Заявки** — заявки с публичного сайта, можно превратить в клиента+проект.
- **Проекты** — CRUD, поиск (название/заказчик/тип/дата/сумма).
- **Заказы** — привязаны к проекту; сумма заказа автоматически берётся из итоговой
  суммы расчёта проекта (Σ A7). Внизу — фиксированная сумма всех заказов.
- **Расчёт** — построчный расчёт A1…A7 по выбранному проекту.
- **Справочник** — типы проектов.
